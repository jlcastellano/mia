Sistemas Expertos - Modelo Clásico

2. Partes de un sistema experto

Cualquier sistema experto típico está compuesto por varios módulos o componentes que trabajan conjuntamente.

Los componentes habituales son los siguientes:

Base de conocimientos

Este componente contiene todo el conocimiento y la experiencia acumulada de los expertos en un campo particular, pero estructurada y codificada de manera que la computadora pueda utilizarla. Esta información puede organizarse mediante diferentes estructuras: listas ordenadas, descripción detallada de objetos y sus propiedades, cálculo de predicados lógicos, redes semánticas que relacionan conceptos, o reglas de producción del tipo "si-entonces".

Base de hechos o datos

Funciona como la memoria de trabajo temporal del sistema. Aquí se almacenan las variables de entrada que proporciona el usuario, las variables intermedias que se van calculando durante el proceso de razonamiento, y las variables de salida que representan las conclusiones. También registra el histórico de estados por los que ha pasado el sistema y sincroniza la información del problema actual con la base de conocimientos para poder deducir nuevos hechos a partir de los que ya se conocen.

Motor o mecanismo de inferencia

Este es el componente lógico central que actúa como el "cerebro" del sistema. Aplica las reglas almacenadas en la base de conocimientos a las consultas o problemas planteados para extraer conclusiones válidas. Determina el orden en que deben ejecutarse las acciones, selecciona qué reglas específicas aplicar en cada momento, y gestiona toda la interacción con el usuario durante el proceso de resolución del problema.

Interfaz sistema-usuario

Es el componente que sirve como vía de comunicación entre el sistema y las personas que lo utilizan. Debe diseñarse para ser natural, sencilla y clara, facilitando que usuarios no técnicos puedan interactuar con el sistema. Habitualmente incluye un módulo de comunicaciones que permite al sistema interactuar con otros sistemas informáticos, y un módulo de explicación que ayuda a verificar la coherencia del proceso de razonamiento y muestra al usuario de manera comprensible cómo se llegó a resolver el problema o a obtener la conclusión.

Módulo de adquisición de conocimiento

Este módulo facilita la tarea de estructurar y actualizar las bases de datos sin que sea necesario tener un perfil técnico muy avanzado o conocimientos de programación. Permite que los expertos del dominio puedan añadir o modificar conocimiento de manera relativamente sencilla, aunque también permite el acceso directo por código para usuarios más técnicos que necesiten hacer modificaciones complejas.
