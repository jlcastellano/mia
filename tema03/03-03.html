Sistemas Expertos - Modelo Clásico

3. Dinámica de un sistema experto

El objetivo fundamental de estos sistemas es lograr que la información crítica del dominio sea explícita y accesible, no enterrada en el código del programa. A diferencia de los programas informáticos tradicionales, donde toda la lógica y las reglas de negocio están incrustadas directamente en el código de programación (lo que dificulta su modificación), en un sistema experto las reglas deben expresarse de manera intuitiva.

Esto permite que los expertos del dominio específico puedan revisar, editar y actualizar esas reglas sin necesidad de ser especialistas en informática o programación. Esta característica facilita enormemente un desarrollo rápido del sistema y un mantenimiento sencillo a lo largo del tiempo.

Mientras que un sistema de base de datos tradicional solo puede recuperar y mostrar información que existe explícitamente almacenada en sus tablas, un sistema experto tiene la capacidad de generar información nueva que no está explícitamente almacenada. Lo hace mediante el razonamiento lógico con los elementos disponibles en su base de conocimientos, actuando efectivamente como un intermediario inteligente entre el conocimiento almacenado y el usuario que necesita respuestas.

3.1. Tipos de sistemas expertos

Existen tres tipos principales de sistemas expertos, cada uno con su propia forma de abordar los problemas:

Basados en reglas (RBR - Rule-Based Reasoning)

Estos sistemas aplican conjuntos de reglas predefinidas y comparan los resultados obtenidos. Frecuentemente utilizan lógica difusa (fuzzy logic) para su evaluación, lo que les permite manejar situaciones donde las respuestas no son simplemente verdadero o falso, sino que tienen grados de certeza.

Basados en casos (CBR - Case-Based Reasoning)

Estos sistemas solucionan problemas nuevos adaptando y reutilizando soluciones que funcionaron para problemas similares en el pasado. Mantienen una biblioteca de casos resueltos y buscan similitudes con el problema actual para proponer soluciones.

Basados en redes bayesianas

Son sistemas de naturaleza probabilística que utilizan el Teorema de Bayes (un principio matemático de probabilidad condicional) para estimar las probabilidades de variables desconocidas. Se basan en un gráfico que representa las dependencias entre diferentes variables del problema.

3.2. Representación del conocimiento

Existen diversas estructuras básicas para representar el conocimiento dentro de un sistema experto:

Reglas de producción

Son estructuras condicionales que siguen el formato "SI se cumple esta premisa, ENTONCES se obtiene esta conclusión o se ejecuta esta acción". Son las más comunes y fáciles de entender.

Lógica proposicional

Esta estructura separa claramente los elementos que constituyen el conocimiento de los mecanismos que controlan cómo opera el sistema, facilitando la modificación independiente de cada parte.

Redes semánticas

Son representaciones visuales similares a mapas conceptuales, donde los conceptos se representan como nodos y las relaciones entre ellos como enlaces o lazos que los conectan.

Marcos (frames)

Son estructuras de datos que asocian propiedades específicas y atributos característicos a cada concepto del dominio, organizando la información de manera jerárquica.

Objetos

Son entidades independientes y autocontenidas utilizadas en entornos de programación orientada a objetos, que encapsulan tanto datos como los métodos que operan sobre esos datos.

Representaciones múltiples

Muchos sistemas expertos combinan varias de estas formas de representación. Pueden ser declarativas (como reglas y redes, que expresan qué es cierto), procedimentales (que expresan cómo hacer algo mediante relaciones explícitas, aunque son más difíciles de modificar), o mixtas que combinan ambos enfoques.

3.3. Mecanismos de razonamiento

Los sistemas expertos utilizan diferentes modos de razonamiento para llegar a conclusiones:

Encadenamiento hacia delante (forward chaining)

Este método parte de los hechos conocidos y aplica reglas sucesivamente para llegar a nuevas conclusiones o resultados. Es útil cuando tenemos datos iniciales y queremos ver qué podemos deducir de ellos.

Encadenamiento hacia atrás (backward chaining)

Este método funciona en dirección opuesta; parte de un resultado o conclusión que queremos verificar y busca retrospectivamente los hechos y reglas que podrían haberlo originado. Es útil para verificar hipótesis.

Búsqueda heurística

Realiza inferencias mediante estructuras de tipo árbol de decisión, explorando diferentes caminos posibles y utilizando reglas heurísticas (reglas prácticas basadas en experiencia) para guiar la búsqueda hacia las soluciones más prometedoras.

Herencia

En sistemas basados en objetos o marcos, un concepto u objeto "hijo" puede adquirir automáticamente propiedades y características de sus conceptos "padres", evitando duplicación de información.

Para obtener conclusiones válidas, estos sistemas emplean sistemas de inferencia lógica básicos como el Modus Ponens (si A implica B, y A es verdadero, entonces B es verdadero) y el Modus Tollens (si A implica B, y B es falso, entonces A es falso).
