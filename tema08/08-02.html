Redes Transformers - Guía Comprensible

2. Los Ingredientes Fundamentales del Transformer

Vamos a construir un Transformer como si preparáramos una receta de cocina, ingrediente por ingrediente.

2.1 Ingrediente #1: Embeddings (Convertir Palabras en Números)

Las computadoras no entienden palabras. Solo entienden números. Así que primero necesitamos convertir cada palabra en un vector de números.

🎨 Analogía: Coordenadas en un Mapa Conceptual

Imagina que cada palabra es un punto en un mapa multidimensional donde la ubicación representa su significado:

Dimensión 1 (Humanidad):    animal ← ... → humano
Dimensión 2 (Edad):         bebé ← ... → anciano  
Dimensión 3 (Género):       masculino ← ... → femenino
... (y 500 dimensiones más)

Entonces:

  • "Rey" podría estar en [0.9 humano, 0.7 adulto, 0.9 masculino, ...]
  • "Reina" podría estar en [0.9 humano, 0.7 adulto, 0.1 femenino, ...]
  • "Perro" podría estar en [0.1 humano, 0.5 adulto, 0.5 neutral, ...]

Matemática famosa: Rey - Hombre + Mujer ≈ Reina

Esto funciona porque los embeddings capturan relaciones semánticas.

Implementación Simple:

import torch
import torch.nn as nn

# Vocabulario simple
vocab = {
    "<PAD>": 0,    # Padding (relleno)
    "el": 1,
    "gato": 2,
    "come": 3,
    "pescado": 4
}

# Tamaño del embedding (cuántas dimensiones)
embedding_dim = 512

# Capa de embedding
embedding_layer = nn.Embedding(
    num_embeddings=len(vocab),  # Tamaño del vocabulario
    embedding_dim=embedding_dim  # Dimensiones del vector
)

# Convertir palabra a número
palabra_id = vocab["gato"]  # 2

# Obtener vector de embedding
vector_gato = embedding_layer(torch.tensor([palabra_id]))
print(f"Vector del gato: {vector_gato.shape}")  # [1, 512]
# Ahora "gato" es un vector de 512 números

¿De dónde vienen estos números?

Al principio, son aleatorios. Durante el entrenamiento, el modelo los ajusta para que palabras similares tengan vectores similares.

2.2 Ingrediente #2: Positional Encoding (Dónde está cada palabra)

Tenemos un problema: los Transformers procesan todas las palabras simultáneamente, pero el orden importa:

"El perro muerde al hombre" ≠ "El hombre muerde al perro"

Solución: Añadir información de posición a cada palabra.

📍 Analogía: Números de Asiento en un Cine

Imagina que cada palabra es una persona en un cine. No basta con saber quiénes están (los embeddings), necesitas saber dónde se sientan:

Fila A: [Persona1, Persona2, Persona3]
        Asiento1  Asiento2  Asiento3

El "Positional Encoding" es como darle a cada persona una etiqueta con su número de asiento. Así, aunque proceses a todas las personas simultáneamente, sabes en qué orden están sentadas.

El Truco Inteligente del Paper Original:

En lugar de usar números simples (1, 2, 3, ...), usan ondas sinusoidales de diferentes frecuencias:

import numpy as np

def positional_encoding(max_len, d_model):
    """
    Crear encodings de posición sinusoidales.
    
    max_len: longitud máxima de la secuencia
    d_model: dimensión del embedding
    """
    # Crear matriz de posiciones
    position = np.arange(max_len)[:, np.newaxis]  # [[0], [1], [2], ...]
    
    # Crear frecuencias diferentes
    div_term = np.exp(np.arange(0, d_model, 2) * -(np.log(10000.0) / d_model))
    
    # Inicializar matriz de encodings
    pe = np.zeros((max_len, d_model))
    
    # Aplicar seno a dimensiones pares
    pe[:, 0::2] = np.sin(position * div_term)
    
    # Aplicar coseno a dimensiones impares
    pe[:, 1::2] = np.cos(position * div_term)
    
    return pe

# Ejemplo de uso
pe = positional_encoding(max_len=100, d_model=512)
print(f"Shape: {pe.shape}")  # (100, 512)

# Visualización simplificada de las primeras 4 posiciones
print("Primeras 4 posiciones (primeros 4 valores):")
print(pe[:4, :4])

¿Por qué seno y coseno?

Porque estas ondas tienen propiedades mágicas:

  • Posiciones cercanas tienen encodings similares
  • El modelo puede "aprender" que posición 5 está cerca de posición 6
  • También puede extrapolar a secuencias más largas de las que vio en entrenamiento

🌊 Analogía: Ondas de Radio

Imagina que cada posición transmite su ubicación como múltiples ondas de radio simultáneamente:

  • Onda lenta (baja frecuencia): Cambia muy gradualmente. Útil para distinguir "principio" vs. "final" de la oración.
  • Onda rápida (alta frecuencia): Cambia rápidamente. Útil para distinguir palabras consecutivas.

Combinando ondas de diferentes velocidades, cada posición tiene una "firma" única pero relacionada con sus vecinos.

2.3 Ingrediente #3: Attention (El Corazón del Transformer)

Esta es la parte más importante y la que hace que todo funcione.

🎯 Analogía: Una Reunión donde Cada Persona Puede Hablar con Cualquier Otra

Imagina una reunión con 10 personas. En lugar de que hablen una por una en orden:

Old School (RNN): Persona 1 habla con Persona 2, quien habla con Persona 3, etc.

Transformer (Attention): Todas las personas están en una mesa redonda. Cada persona puede:

  1. Hacer preguntas a todos los demás (Query: "¿Quién tiene información sobre X?")
  2. Responder con su información (Key: "Yo tengo información sobre Y")
  3. Compartir detalles cuando se lo piden (Value: "Aquí están los detalles...")

Cada persona decide A QUIÉN escuchar más atentamente basándose en qué tan relevantes son sus respuestas.

2.3.1 Los Tres Componentes: Query, Key, Value

Vamos a entender esto con un ejemplo práctico:

Oración: "El gato negro duerme en la cama"

Queremos que la palabra "duerme" entienda su contexto. ¿Quién duerme? ¿Dónde?

Query (Pregunta): "duerme" pregunta: "¿Quién es el sujeto? ¿Dónde ocurre?"

Keys (Etiquetas): Cada palabra dice qué información ofrece:

  • "El" → "Soy un artículo"
  • "gato" → "Soy un animal (sujeto)"
  • "negro" → "Soy un adjetivo"
  • "en" → "Soy una preposición de ubicación"
  • "cama" → "Soy un lugar"

Values (Contenido): El contenido real de cada palabra

El Proceso:

  1. "duerme" compara su Query con todas las Keys
  2. Descubre que "gato" y "cama" son muy relevantes
  3. Presta MUCHA atención a esos valores
  4. Presta POCA atención a "el", "negro", "en"

Resultado: "duerme" ahora tiene una representación enriquecida que sabe que "el gato duerme en la cama".

2.3.2 Implementación Simplificada de Attention

import torch
import torch.nn.functional as F

def simple_attention(query, key, value):
    """
    Implementación básica de attention.
    
    query: [batch, seq_len, d_k] - Las preguntas
    key:   [batch, seq_len, d_k] - Las etiquetas
    value: [batch, seq_len, d_v] - Los contenidos
    """
    
    # Paso 1: Calcular similitud entre queries y keys
    # ¿Qué tan similar es cada pregunta con cada etiqueta?
    d_k = query.size(-1)
    scores = torch.matmul(query, key.transpose(-2, -1))  # [batch, seq_len, seq_len]
    
    # Paso 2: Escalar (para estabilidad numérica)
    scores = scores / torch.sqrt(torch.tensor(d_k, dtype=torch.float32))
    
    # Paso 3: Softmax (convertir a probabilidades)
    # Esto convierte los scores en "pesos de atención"
    attention_weights = F.softmax(scores, dim=-1)
    
    # Paso 4: Aplicar pesos a los valores
    # Cuanto mayor el peso, más información tomamos de ese valor
    output = torch.matmul(attention_weights, value)
    
    return output, attention_weights

# Ejemplo de uso
batch_size = 1
seq_len = 5  # "el gato negro duerme en"
d_model = 64

# Simular embeddings
embeddings = torch.randn(batch_size, seq_len, d_model)

# Por simplicidad, usar mismo embedding para Q, K, V
# (En la práctica, se proyectan a través de matrices diferentes)
output, weights = simple_attention(embeddings, embeddings, embeddings)

print(f"Output shape: {output.shape}")  # [1, 5, 64]
print(f"Attention weights shape: {weights.shape}")  # [1, 5, 5]

# Visualizar pesos de atención para la palabra 3 (duerme)
print("\nPesos de atención de la palabra 3:")
print(weights[0, 3, :])  # ¿A qué palabras presta atención?

2.3.3 Multi-Head Attention (Múltiples Perspectivas)

Un solo mecanismo de atención puede perderse detalles. Es como mirar una pintura desde un solo ángulo.

👥 Analogía: Múltiples Expertos Analizando

Imagina que tienes 8 expertos diferentes analizando la misma oración:

  • Experto 1 (sintáctico): Se enfoca en relaciones gramaticales. Conecta sustantivos con verbos, adjetivos con sustantivos.
  • Experto 2 (semántico): Se enfoca en significado. Conecta "gato" con "animal", "duerme" con "cama".
  • Experto 3 (temporal): Se enfoca en secuencia temporal de eventos.
  • Experto 4 (causal): Se enfoca en relaciones causa-efecto.
  • ...y 4 expertos más, cada uno buscando patrones diferentes.

Al final, combinas las observaciones de los 8 expertos para tener una comprensión completa.

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        
        assert d_model % num_heads == 0, "d_model debe ser divisible por num_heads"
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # Dimensión por cabeza
        
        # Proyecciones para crear Q, K, V
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        
        # Proyección final
        self.W_o = nn.Linear(d_model, d_model)
    
    def split_heads(self, x):
        """Dividir en múltiples cabezas"""
        batch_size, seq_len, d_model = x.size()
        # Reshape a [batch, seq_len, num_heads, d_k]
        x = x.view(batch_size, seq_len, self.num_heads, self.d_k)
        # Transponer a [batch, num_heads, seq_len, d_k]
        return x.transpose(1, 2)
    
    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)
        
        # 1. Proyectar Q, K, V
        Q = self.W_q(query)  # [batch, seq_len, d_model]
        K = self.W_k(key)
        V = self.W_v(value)
        
        # 2. Dividir en múltiples cabezas
        Q = self.split_heads(Q)  # [batch, num_heads, seq_len, d_k]
        K = self.split_heads(K)
        V = self.split_heads(V)
        
        # 3. Aplicar attention a cada cabeza
        d_k = self.d_k
        scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(d_k, dtype=torch.float32))
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attention_weights = F.softmax(scores, dim=-1)
        attention_output = torch.matmul(attention_weights, V)
        
        # 4. Concatenar cabezas
        # [batch, num_heads, seq_len, d_k] → [batch, seq_len, num_heads, d_k]
        attention_output = attention_output.transpose(1, 2).contiguous()
        # → [batch, seq_len, d_model]
        attention_output = attention_output.view(batch_size, -1, self.d_model)
        
        # 5. Proyección final
        output = self.W_o(attention_output)
        
        return output

# Uso
mha = MultiHeadAttention(d_model=512, num_heads=8)
x = torch.randn(1, 10, 512)  # [batch, seq_len, d_model]
output = mha(x, x, x)
print(f"Output shape: {output.shape}")  # [1, 10, 512]

2.4 Ingrediente #4: Feed-Forward Networks (Procesamiento Individual)

Después de que cada palabra ha recopilado información de todas las demás a través de attention, necesita "pensar" individualmente sobre esa información.

🧠 Analogía: Reflexión Personal

Imagina que después de la reunión (attention), cada persona se va a su oficina privada a procesar toda la información que escuchó y formar sus propias conclusiones.

Attention: "Escuchar a todos"
Feed-Forward: "Pensar en lo que escuché"

Es una red neuronal simple aplicada a cada posición independientemente.

class FeedForward(nn.Module):
    def __init__(self, d_model=512, d_ff=2048, dropout=0.1):
        super().__init__()
        
        # Dos capas lineales con ReLU en medio
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)
    
    def forward(self, x):
        # x: [batch, seq_len, d_model]
        
        # Expandir
        x = self.linear1(x)  # [batch, seq_len, d_ff]
        x = F.relu(x)
        x = self.dropout(x)
        
        # Contraer de vuelta
        x = self.linear2(x)  # [batch, seq_len, d_model]
        
        return x

# Uso
ff = FeedForward(d_model=512, d_ff=2048)
x = torch.randn(1, 10, 512)
output = ff(x)
print(f"Output shape: {output.shape}")  # [1, 10, 512]

¿Por qué expandir y contraer?

Es como comprimir y descomprimir información:

  • Expandir (512 → 2048): Crea más espacio para representaciones ricas e intermedias
  • ReLU: Introduce no-linealidad (permite aprender patrones complejos)
  • Contraer (2048 → 512): Destila la información de vuelta al tamaño original

2.5 Ingrediente #5: Residual Connections y Layer Normalization

Estos son trucos cruciales que hacen que las redes profundas (muchas capas) funcionen.

2.5.1 Residual Connections (Conexiones de Atajo)

🏃 Analogía: Atajos en un Edificio

Imagina un edificio de 10 pisos. Tienes que subir desde el piso 1 al piso 10:

Sin atajos: Subes piso por piso. Si hay un problema en el piso 5, la información del piso 1 podría no llegar bien al piso 10.

Con atajos (residual connections): Además de las escaleras, hay un ascensor directo. La información puede fluir directamente del piso 1 al 10, Y también puede tomar las escaleras si aprende algo útil en el camino.

Matemáticamente:

Output = Input + SubLayer(Input)

No reemplazamos el input, lo MEJORAMOS.

class TransformerBlock(nn.Module):
    def __init__(self, d_model=512, num_heads=8, d_ff=2048, dropout=0.1):
        super().__init__()
        
        self.attention = MultiHeadAttention(d_model, num_heads)
        self.feed_forward = FeedForward(d_model, d_ff, dropout)
        
        # Layer Normalization
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        
        self.dropout1 = nn.Dropout(dropout)
        self.dropout2 = nn.Dropout(dropout)
    
    def forward(self, x, mask=None):
        # Attention con residual connection
        attn_output = self.attention(x, x, x, mask)
        x = x + self.dropout1(attn_output)  # ← Residual connection
        x = self.norm1(x)  # ← Normalization
        
        # Feed-forward con residual connection
        ff_output = self.feed_forward(x)
        x = x + self.dropout2(ff_output)  # ← Residual connection
        x = self.norm2(x)  # ← Normalization
        
        return x

2.5.2 Layer Normalization

⚖️ Analogía: Normalizar Calificaciones

Imagina que tienes tres profesores que califican de forma diferente:

  • Profesor A: Da calificaciones de 60-70 (difícil)
  • Profesor B: Da calificaciones de 80-90 (fácil)
  • Profesor C: Da calificaciones de 40-100 (muy variable)

Para comparar estudiantes, normalizas las calificaciones de cada profesor:

  • Calculas el promedio y desviación estándar de cada profesor
  • Ajustas todas las calificaciones para que tengan la misma escala

Layer Normalization hace lo mismo con las activaciones de la red: las mantiene en un rango estable para que el entrenamiento sea más fácil.

class LayerNorm(nn.Module):
    def __init__(self, d_model, eps=1e-6):
        super().__init__()
        # Parámetros aprendibles
        self.gamma = nn.Parameter(torch.ones(d_model))
        self.beta = nn.Parameter(torch.zeros(d_model))
        self.eps = eps
    
    def forward(self, x):
        # x: [batch, seq_len, d_model]
        
        # Calcular media y varianza en la última dimensión
        mean = x.mean(dim=-1, keepdim=True)
        std = x.std(dim=-1, keepdim=True)
        
        # Normalizar
        x_norm = (x - mean) / (std + self.eps)
        
        # Escalar y desplazar (parámetros aprendibles)
        return self.gamma * x_norm + self.beta
